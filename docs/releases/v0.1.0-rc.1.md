# z2000 0.1.0-rc.1

This is the first public release candidate of z2000, an inspectable
JPEG2000 Part 1 codec and TIFF-to-JP2 conversion tool written in Zig.

## Highlights

- Lossless RGB and grayscale TIFF-to-JP2 conversion for uncompressed chunky
  8-bit and 16-bit strips.
- Reversible RCT/5-3 and irreversible ICT/9-7 paths with strict no-sidecar
  decode.
- Single- and multi-tile coding, all five progression orders, quality layers,
  global rate allocation, PLT/TLM, POC, PPM/PPT, and the six Part 1 code-block
  style bits in their documented profile envelope.
- Optional ICC profile preservation without implicit pixel conversion.
- OpenJPEG, Grok, Kakadu, and jpylyzer interoperability gates for maintained
  profiles.
- Portable Zig vector kernels and multi-threaded encode/decode.

## Release Candidate Policy

The CLI and internal Zig API are not stable before 1.0. Unsupported profile
combinations fail closed, but this release is not a general JP2/JPX decoder.
The project's 100/100 ISO engineering scorecards are internal coverage tools,
not formal ISO conformance certification.

Known boundaries include uncompressed strip-based TIFF input, bounded
component layouts, no JPX feature set, opaque ICC preservation rather than
color conversion, and no JPEG/PNG/BMP/RAW/OpenEXR conversion front ends yet.
See `README.md` and `docs/iso_coverage.md` for the exact supported envelope.

The downloadable binaries are currently unsigned. Verify the archive against
the accompanying `SHA256SUMS` file before use.
