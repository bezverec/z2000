# z2000 0.2.0-rc.1

This release candidate broadens z2000's checked JPEG2000/JP2 profiles and adds
bounded raster-to-JP2 conversion front ends while keeping unsupported input
fail-closed.

## Highlights Since 0.1.0-rc.1

- Added component-generic one- through four-plane paths, gray/RGB alpha
  semantics, mixed 8/16-bit precision, sampled component geometry, packed
  packet headers, and bounded sampled reversible encoding with quality layers,
  multi-tile layouts, and checked POC schedules.
- Added explicit sYCC conversion, bounded ICC v2/v4 RGB matrix/TRC conversion,
  and plane-preserving signalling for CMYK, default CIELab, e-sRGB, and e-sYCC.
- Added bounded BMP, PNG, baseline JPEG, LinearRaw DNG, and normalized-linear
  OpenEXR input adapters. Extension-inferred single-file commands and sorted,
  non-recursive wildcard batch conversion are available through `z2k`.
- Added byte-exact EXIF, XMP, and IPTC-IIM preservation from the bounded JPEG
  profile into checked JP2 UUID boxes.
- Capped and pooled the reversible 5/3 worker phase while retaining
  deterministic output across thread counts.

## Release Candidate Policy

The CLI and Zig API remain unstable before 1.0. This is not a general JP2/JPX,
camera RAW, or HDR converter. Each input adapter has a deliberately narrow
documented envelope; progressive/arithmetic JPEG, Adam7 PNG, CFA/demosaicing,
compressed or tiled EXR, general ICC transforms, extended XMP, and semantic
metadata rewriting remain unsupported. JP2 metadata payloads are preserved but
are not yet restored into TIFF output.

The project's 100/100 ISO engineering scorecards are internal coverage tools,
not formal ISO conformance certification. The binaries are unsigned; verify
the platform archive against `SHA256SUMS`. Every archive contains identical
`z2000` and `z2k` executables for the long and short command names.
